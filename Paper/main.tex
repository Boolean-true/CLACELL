%\subsection{The Challenge}
Despite the rapid evolution of scRNA-seq technologies, the classification of cell types remains a significant analytical bottleneck. Historically, cell-type annotation has relied heavily on manual marker-based approaches like scumi \cite{ding_systematic_2020}. This traditional method requires the manual specification of the marker genes. However, manual annotation is highly subjective, labor-intensive, and therefore scale-limited, making it impractical for modern, atlas-level datasets \cite{abdelaal_comparison_2019}.

To overcome these limitations, numerous classification algorithms were designed that learn from well-annotated reference datasets and transfer these labels to unannotated query datasets. While promising, many of these existing classifiers show substantial drawbacks when deployed on complex tissues like the immune system. Comprehensive benchmarks revealed that these tools frequently suffer from a lack of robustness, yielding poor F1 scores, high variance in classification accuracy, and a high sensitivity to gene dropout \cite{abdelaal_comparison_2019}. %Furthermore, some methods rely on non-deterministic processes, complicating reproducibility. %TODO: Quelle finden um das zu belegen

%An additional layer of complexity arises from the current software ecosystem. A substantial portion of early foundational single-cell preprocessing and normalization tools, including the \texttt{scumi} pipeline \cite{ding_systematic_2020}, were developed in R. In contrast, the modern ecosystem for scalable machine learning and deep learning is predominantly situated within Python, anchored by frameworks such as \texttt{Scanpy} \cite{Wolf2018}, \texttt{scikit-learn}, and \texttt{PyTorch}. This language barrier creates a transition gap that hinders the seamless integration of robust preprocessing routines with advanced, state-of-the-art predictive modeling.

CLACELL bridges the gap between specialized bioinformatics preprocessing and high-performance machine learning, providing an end-to-end scalable framework for cell-type annotation. %TODO: Letzten Satz überarbeiten; scalable evtl mit reinbringen

This suggests that SingleR has a strong tendency to overfit on the training data. Consequently, it struggles to generalize to unseen data with different underlying distributions or batch effects, which limits its reliability for the zero-shot annotation of new datasets.

In contrast, scGPT demonstrates a strong performance on the out-of-distribution test set and maintains consistency across varying dropout rates. \subsection{Limitations}\label{sec:discussion:limitations}

Despite the promising results of our study, there are several limitations that should be acknowledged. First, the labels for the training of our classifiers were provided using a marker-based annotation approach. Therefore the quality of the labels is dependent on the quality of the markers. 
Second, we only used a single dataset for training and evaluation, which may limit the generalizability of our findings to other datasets or cell types.
Third, the labels are coarse as the markers from scumi are not very specific. In order to get more specific labels, more fine-grained markers are needed.
Finally, our proposed classifier has a low performance on the Natural Killer cells in the out-of-distribution test set. Therefore it cannot be used for the accurate annotation of this cell type on new datasets.

\subsection{Conclusion \& Future Work}\label{sec:discussion:conclusion}

In this paper, we presented a robust cell type classifier for immune cells based on traditional machine learning methods. Our results demonstrate that our proposed classifier outperforms existing state-of-the-art methods in terms of accuracy and robustness. We also showed that our classifier is able to generalize well to out-of-distribution datasets. It has a low variance, is reproducible and is resource-efficient, requiring only a CPU and approximately 14 GB of RAM for training.

As future work, we plan to classify datasets into more fine-grained cell types. In order to do this, we can either use more specific markers or use a hierarchical classification approach where we add a sub classifier that classifies the cells from coarse classes into finer classes. Additionally, we want to adress the low performance on the Natural Killer cells. This could be done by adding more samples of this cell type to the training dataset or by combining different classifiers in our custom ensemble.

Another potential improvement is to add a rejection class to the classifier. This would allow the classifier to reject samples that it is not confident about, which could increase the overall performance and robustness of our classifier.

\subsection{Machine Learning Algorithms}\label{sec:methods:machine-learning}

To classify the immune cells based on their gene expression profiles, we evaluated several machine learning models, categorized into linear and tree-based algorithms. For the linear approaches, we implemented Logistic Regression and a Linear Support Vector Classifier (LinearSVC). For tree-based models, we used Random Forest, Extra Trees and LightGBM.

The hyperparameter search spaces for each model are detailed in \Cref{tab:ml_hyperparameters}. To ensure compatibility with different loss functions and improve convergence, we used the saga solver with a maximum of 1,000 iterations for Logistic Regression for all iterations. LightGBM's objective was fixed to multiclass for our evaluation, as it is a multiclass classification problem. Furthermore, the number of parallel jobs was set to 1 as the hyperparameter tuning algorithm already parallelizes the training of the models.

Hyperparameter tuning was performed using Bayesian Optimization via \texttt{BayesSearchCV} from the scikit-optimize library \cite{scikit-optimize_contributors_skoptbayessearchcv_2020} with 30 iterations and a 5-fold cross-validation. Compared to traditional grid or random search strategies, Bayesian optimization offers enhanced efficiency by using past evaluation results to find better parameter choices, thereby finding optimal configurations faster. \cite{NIPS2012_05311655}

To optimize computational resources, a custom early-stopping mechanism was integrated into the optimization. This mechanism terminates the search if the model's performance failed to improve by more than 0.2 \% over 5 consecutive iterations. To ensure sufficient initial exploration of the hyperparameter space, this stopper was restricted from triggering until the 15th iteration. Consequently, if no significant improvement was observed between iterations 10 and 15, the search was terminated immediately at the 15th iteration, preventing unnecessary computational overhead. %This approach substantially reduced computational overhead without compromising the quality of the final hyperparameter configurations. %TODO: Diesen Satz noch schreiben?

The best Machine Learning models were combined into a custom feature-subsampling voting ensemble. This ensemble approach was designed to enhance robustness by training multiple instances of the same model on different subsets of features. The feature subsets include the full set of features and three additional subsets where the top features -- determined by a feature importance ranking -- were systematically removed. Specifically, we created subsets by excluding the top 0.75~\%, 1.5~\%, and 2.5~\% of features based on their importance scores.
For the linear models we used the feature importance of the random forest model, while for the tree-based models we used the feature importance of the logistic regression model. 
By using the feature importance of a different machine learning approach, we avoid the bias of the feature importance ranking towards the model that is used for the ensemble. Furthermore the insights from two different approaches are combined to improve the overall performance.

We used a soft voting strategy for the ensemble, where the predicted probabilities of each base model are averaged to make the final prediction. This approach takes the confidence of each base model in its predictions into account, aiming to improve robustness and accuracy.

To save computational resources, we used the optimal hyperparameters from the tuning of the single models for the ensemble.

Following the baseline evaluation, we introduce three distinct robustness tests to challenge the models:

\begin{enumerate}
    \item \textbf{Random Dropout:} We simulate noisy data by randomly selecting 10 \% of the input features and setting their values to 0. This test quantifies how the well the model can handle noisy data and how its performance degrades when faced with missing features.
    \item \textbf{Prediction Stability:} To evaluate prediction consistency under stochastic inference, we execute predictions on the same dataset across five independent runs. The number of inconsistent predictions across these iterations is aggregated to measure the stability and variance of the models.
    \item \textbf{Feature Importance Dropout:} The final and most challenging test evaluates model resilience against the loss of highly informative biological signals. We progressively remove the most critical features by setting their values to 0 at four distinct thresholds: 0.1 \%, 0.5 \%, 1 \% and 2 \% of the total feature space. The feature importance for the Dropout was determined primarily by the marker genes identified by scumi. Since the number of these marker genes was insufficient to meet the higher percentage thresholds, the remaining top features were supplemented via differential gene expression analysis computed on the training set. This test reveals how heavily each model relies on primary marker genes and how robustly they utilize alternative features when key biological markers are missing. It also assesses how the performance of each model degrades as the proportion of removed important features increases.
\end{enumerate}

\subsection{Classification}\label{sec:package:classification}

The Classifier \texttt{CellClassifier} from the CLACELL package implements the custom ensemble with the LinearSVC. It provides four methods:

\begin{itemize}
    \item \texttt{def grid\_search(X\_train, y\_train, X\_test, y\_test, labels, n\_jobs)}: This method makes a random search with 50 iterations to find the optimal hyperparameters for the LinearSVC. Afterwards, it calls the train method with the optimal hyperparameters and the combined training and test data. It takes as input the features \texttt{X} and corresponding labels \texttt{y} for the training and test sets, the field where the labels are stored and the number of jobs for parallel processing. %TODO: Anpassen, wenn ich die Funktion anpasse!
    \item \texttt{train(X\_train, y\_train, **hyperparameters)}: This method trains the custom ensemble with the given hyperparameters on the given data.     \item \texttt{evaluate(X\_test, y\_test, labels, X\_ood, y\_ood, feature\_importances, log\_to\_console, log\_to\_file)}: This method evaluates the model's performance on a in distribution and out of distribution test set. It is a wrapper around the tests presented in \Cref{sec:methods:robustness-tests}. It can only be called if the model has been trained. It returns a multi index dataframe with the robustness report.
    \item \texttt{predict(X)}: This method predicts the cell types for the given input features \texttt{X}. It can only be called if the model has been trained. It returns the predicted labels.
