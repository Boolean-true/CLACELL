\subsection{Ensemble Learning Strategies}\label{sec:methods:ensemble-learning}

The best Machine Learning models were combined into ensemble learning strategies to improve the overall performance and robustness of the classification. We used standard voting and stacking classifiers, as well as a custom feature-subsampling voting ensemble. In the custom ensemble, we combined the same machine learning model mutliple times, each time trained on a different subset of features. The subsets of features include the full set of features and three subsets of features where the top features according to a feature importance ranking were removed. This allows the ensemble to be more robust to missing features and to better generalize to new data.
For the linear models we used the feature importance of the random forest model, while for the tree-based models we used the feature importance of the logistic regression model. %TODO: Dann muss ich noch Random Forest damit machen, oder umschreiben
By using the feature importance of a different machine learning approach, we avoid the bias of the feature importance ranking towards the model that is used for the ensemble. Furthermore the learnings from two different approaches are combined to improve the overall performance. %TODO: Evtl. noch besser erklären

\subsection{Robustness Tests}\label{sec:methods:robustness-tests}

The robustness tests that are performed to evaluate the models contain two parts with the same tests: first the models are evaluated on the in-distribution test set, which is a subset of the training dataset that was not used for training. %TODO: Passt das mit dem :?
To avoid data leakage, the split is performed on the donors, so that the donors in the training set are not present in the test set. The donors for the test set were chosen so that the the samples in the test set is close to 30% of the whole dataset. %TODO: hier erwähnen oder woanders?; Hier auch schonmal erwähnen, dass wenig Testdaten für zwei Klassen?
After the in-distribution tests, the models are evaluated on an out-of-distribution test set, which contains data from a different source than the training dataset. 
%The performance of the models on the out-of-distribution test set is compared to their performance on the in-distribution test set to assess their robustness.
By comparing the performance of the models between an in-distribution and an out-of-distribution test set, we can better assess their robustness on new data. %TODO: Mehr zu dem ID OOD Vergleich schreiben?

The first robustness test for each distribution is the baseline score on the whole dataset. The first metric for that is accuracy, which gives a first impression of the performance of the models. However, since the dataset is imbalanced, the accuracy alone is not sufficient to assess the performance of the models. Therefore, we also compute the classification report, which contains the precision, recall, F1-score and support for each class as well as the macro and weighted averages. This allows us to assess the performance of the models on each class individually. % TODO: sowas erwähnen? The classification report is computed using the \texttt{classification\_report} function from the \texttt{sklearn.metrics} module.

After assessing the baseline performance of the models, we perform a robustness test by adding noise to the input data. The noise is added to the input data by randomly selecting 10 \% of the features and replacing their values with 0. This allows us to assess how well the models can handle noisy data and how their performance degrades with missing features.

Another robustness feature the models are tested on is the number of inconsistent predictions when predicting the same data multiple times. We perform this test by predicting the same data 5 times and counting the number of inconsistent predictions over all runs. This allows us to assess how stable the models are and how much their predictions vary with different runs. %TODO: Doppelt sich da nicht was im letzten Satz?; evtl besser begründen

The last robustness test is the most challenging one, as it removes the most important features from the input data and computes the accuracy of the model. The most important features are determined by the marker genes that were obtained by scumi. However, since there aren't enough marker genes, the rest of the important features are determined by a differential gene expression on the training dataset. %TODO: Besser erklären
The feature importance dropout test is performed four times with different percentages of the most important features removed: 0.1 \%, 0.5 \%, 1 \% and 2 \%. Again the values of the removed features are set to 0. This allows us to assess how well the models can handle missing important features and how their performance degrades with increasing feature dropout. It also shows how much each model relies on the most important features for its predictions.
